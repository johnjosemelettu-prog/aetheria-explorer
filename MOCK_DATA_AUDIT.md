}]# Mock Data Audit and Live Release Actions

This document identifies all areas of the Aetheria Explorer application where mock data is currently used. It also outlines the necessary actions to replace this data with live, production-ready implementations before the application is released.

---

## 1. `src/components/AIFeatures.tsx`

- **Mock Data Identified**:
    - `mockItinerary`: A hardcoded itinerary object for a 3-day trip to Paris is used to demonstrate various AI-powered itinerary manipulation features.
    - `mockExpenses`: A sample array of expense objects is used to showcase the shared expense synthesis feature.
    - `mockSouvenirItems`: A hardcoded array of souvenir item IDs is used for a conceptual feature.

- **Actions Required Before Live Release**:
    - **`mockItinerary`**: This should be replaced with a dynamic data flow. The component should receive the itinerary from the main AI Itinerary Synthesizer or a state management store. The AI features will then operate on the actual, user-generated itinerary.
    - **`mockExpenses`**: A new UI/feature needs to be developed to allow users to input their own expenses. The data from this new feature will then be fed into the `synthesizeSharedExpenses` function.
    - **`mockSouvenirItems`**: This conceptual feature needs to be connected to the user's actual profile and the Quantum Souvenir system. The component should fetch the user's collected souvenirs from the database.

---

## 2. `src/components/Store.tsx`

- **Mock Data Identified**:
    - `mockProducts`: The component uses a hardcoded array of `Product` objects to populate the store. The image URLs are sourced from `unsplash.com`, a placeholder service.

- **Actions Required Before Live Release**:
    - **Backend Integration**: A backend service and database are required to manage the products. This includes creating, reading, updating, and deleting product information.
    - **API Endpoint**: An API endpoint must be created to fetch the list of products.
    - **Component Update**: The `useEffect` hook in the `Store` component must be updated to call this new API endpoint instead of using the `mockProducts` array. The loading and error states should be handled appropriately.

---

## 3. General AI Skill Calls in `AIFeatures.tsx`

- **Mock Data Identified**:
    - Many of the AI function calls in the `useEffect` hook of `AIFeatures.tsx` use hardcoded strings and values as inputs (e.g., `AI.findSkillExchange("Barcelona")`, `AI.getTravelerHeatmap("Kyoto")`).

- **Actions Required Before Live Release**:
    - **Dynamic Inputs**: These hardcoded inputs should be replaced with dynamic values derived from the user's context, the generated itinerary, or user-configurable settings. For example, the destination for the traveler heatmap should be taken from the current itinerary's destination.
